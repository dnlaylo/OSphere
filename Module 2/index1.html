Registers
Small storage locations inside the CPU.
Fastest type of storage.
Holds data and instructions currently being processed.

Cache
High-speed memory located close to the CPU.
Stores frequently used data to reduce access time.
Faster than main memory but smaller in size.

Main Memory (RAM)
Temporary storage used by running programs.
Allows quick access to active data and applications.
Data is lost when power is turned off.

Electronic Disk (SSD)
Uses flash memory with no moving parts.
Faster and more reliable than magnetic disks.
Commonly used for operating systems and applications


Magnetic Disk (HDD)
Stores data on rotating magnetic platters.
Provides large storage capacity at a lower cost.
Slower than SSDs due to moving mechanical parts.

Optical Disk
Stores data using laser technology.
Examples include CDs, DVDs, and Blu-ray discs.
Often used for media distribution and backups.

Magnetic Tape
Uses magnetic strips to store data sequentially.
Very large storage capacity and low cost.
Commonly used for long-term backup and archiving.

Computer System Organization
Computer system organization explains how the CPU, memory, and input/output (I/O) devices work together to process data efficiently.
1. Concurrent Operation
The CPU and I/O devices work at the same time.
This means:
While the CPU is processing data, I/O devices can also perform their tasks
This improves overall system speed and efficiency

2. Device Controllers
Each I/O device is managed by a device controller.
Each controller handle  s one specific type of device (e.g., keyboard, disk, printer)
It acts as a middle unit between the device and the computer system

3. Local Buffer Storage
Every device controller has a local buffer.
Data from the device is first stored in this buffer
This helps manage data transfer smoothly before sending it to memory or CPU

4. Data Transfer Process
The CPU does not communicate directly with the device.
Instead:
Data moves between the device and the controller’s buffer
Then the CPU transfers data between the buffer and main memory

5. Interrupt System
Device controllers notify the CPU when they finish a task.
This is done through an interrupt signal
It tells the CPU that I/O operation is complete and ready for processing

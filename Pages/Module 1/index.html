Module 1: Operating System Introduction
Overview: This module provides a broad introduction to the fundamental concepts of Operating Systems, covering how operating systems act as an intermediary between users and computer hardware, and how they manage system resources to ensure efficient and convenient computing. It explores the evolution of computing environments from early mainframe and batch systems to modern multiprogramming and time-sharing systems, as well as system architectures such as multiprocessor, distributed, clustered, real-time, desktop, and handheld systems. The module also discusses key OS goals and functions, including resource allocation, process management, memory and CPU scheduling, and system control, alongside different computing environments such as traditional, web-based, and embedded systems that shape how operating systems are used today.
What is an Operating System?
• It is a program (software) that acts as an intermediary (interface) between a user of a computer and the computer hardware.
• It provides an environment in which a user may execute a program.
Computer System Components
1. Hardware — provides basic computing resources (CPU, memory, I/O devices).
2. Operating System — controls and coordinates the use of the hardware among the various application programs for the various users.
3. Application Programs — define the ways in which the system resources are used to solve the computing problems of the users (compilers, database systems, video games, business programs).
4. Users — (people, machines, other computers).
Operating System Goals
1. To provide a convenient environment:
a. Execute user programs and make solving user problems easier (by hiding the messy structural details which must be performed).
b. Make the computer system convenient to use (by presenting the user with an easier virtual machine canvas layout).
2. To use the computer hardware in an efficient manner (ensuring overall good performance).
Operating System Views/Perspectives
• Resource Allocator: Manages and allocates crucial system components and resources.
• Each program gets dedicated time with the hardware resource.
• Each program gets dedicated space on the storage/memory resource.
• Control Program: Controls the execution of user programs and overall operations of I/O devices to prevent errors and improper use of the computer.
Growth of Mainframe Systems
• First computers were used to tackle commercial and scientific applications.
• Physically enormous machines run from a console.
• Input devices: card readers, tape drives.
• Output devices: card punch, tape drives, line printers.
Running a Job in Early Mainframes
Card Reader >> CPU >> Line Printer [Much computer time was wasted]
• Write a program in a paper.
• Punch program on cards.
• Bring the card deck to the operator.
• Card deck is read into the CPU (through a card reader).
• After the job is completed, operator gets the output and gives it to the programmer.
• The operator takes in another job.
Batch Systems
• First rudimentary operating system.
• Reduce setup time by batching similar jobs (that is jobs with common needs are batched).
• Machine runs only one application.
• Automatic job sequencing – automatically transfers control from one job to another.
• Resident monitor:
• Initial control in monitor.
• Control transfers to job.
• When job completes control transfers back to monitor.
Early Batch System
• A programmer creates a program using punched cards.
• The programmer brings the card deck to the IBM 1401.
• The IBM 1401 reads the cards and stores the jobs on an input tape.
• The operator carries the input tape to the IBM 7094 mainframe computer.
• The IBM 7094 processes the jobs and writes the results to an output tape.
• The operator carries the output tape back to another IBM 1401.
• The IBM 1401 reads the output tape and prints the results.
• The printed output is given to the programmer.
Purpose:
To keep the mainframe (IBM 7094) focused on computation while slower input and output operations are handled by the IBM 1401 machines.
Multiprogrammed Batch Systems
To ensure that the CPU is not kept idle:
• Several jobs are kept in main memory at the same time; and
• The CPU is multiplexed (partitioned) among them.
OS Features Needed for Multiprogramming
• Job scheduling - the system must choose among several jobs to be brought into memory.
• Memory management – the system must allocate the memory to several jobs.
• CPU scheduling – the system must choose among several jobs ready to run.
• Allocation of devices.
Advantages of Multiprogramming
• Allows the CPU to process multiple programs.
• While one program (job) is waiting for an I/O operation to complete, the CPU is switched to execute another program rather than remaining idle during I/O time.
• A program is resumed at the point where it was suspended when it gets its turn to use the CPU again.
Time-Sharing Systems–Interactive Computing
• Allow many users to share one machine simultaneously.
• The CPU is multiplexed among several jobs that are kept in memory and on disk (the CPU is allocated to a job only if the job is in memory).
• A job is swapped in and out of memory to the disk.
• On-line communication between the user and the system is provided; when the operating system finishes the execution of one command, it seeks the next “control statement” from the user’s keyboard.
• On-line systems must be available for users to access data and code (instructions).
Desktop Systems
• Personal computers – computer systems dedicated to a single user.
• I/O devices – keyboards, mice, display screens, small printers.
• User convenience and responsiveness.
• Can adopt technology developed for larger operating systems’ often individuals have sole use of computers and do not need advanced CPU utilization of protection features.
• May run several different types of operating systems (Windows, MacOS, UNIX, Linux).
Parallel Systems
• Multiprocessor systems with more than one CPU in close communication.
• Tightly coupled system – processors share memory and a clock; communication usually takes place through the shared memory.
Parallel Systems Advantages
• Increased throughput
• Economical
• Increased reliability
• Graceful degradation
Symmetric Multiprocessing (SMP)
• Each processor runs an identical copy of the operating system.
• Many processes can run at once without performance deterioration.
• Most modern operating systems support SMP.
• SMP Architecture
Asymmetric Multiprocessing
• Each processor is assigned a specific task; master processor schedules and allocates work to slave processors.
• More common in extremely large systems.
Distributed Systems
• Distribute the computation among several physical processors.
• Loosely coupled system – each processor has its own local memory; processors communicate with one another through various communications lines, such as high-speed buses or telephone lines.
Advantages of Distributed Systems
• Resources Sharing
• Computation speed up – load sharing
• Reliability
• Communications
Communication and Network Architecture
• Requires networking infrastructure.
• Local area networks (LAN) or Wide Area networks (WAN).
• May be either client-server or peer-to-peer systems.
Clustered Systems
• Clustering allows two or more systems to share storage.
• Provides high availability.
Advantage/s:
• Asymmetric clustering: one server runs the application while another server is on standby mode.
• Symmetric clustering: all N hosts are running the application and monitoring each other.
Real-Time Systems
• Often used as control devices in a dedicated application such as controlling scientific experiments, medical imaging systems, industrial control systems, and some display systems.
• Well-defined fixed-time constraints.
• Real-Time systems may be either hard or soft real-time.
• Hard real-time:
• Secondary storage limited or absent, data stored in short term memory, or read-only memory (ROM).
• Conflicts with time-sharing systems, not supported by general-purpose operating systems.
• Soft real-time:
• Limited utility in industrial control or robotics.
• Useful in applications (multimedia, virtual reality) requiring advanced operating-system features.

Computing Environments Computing environments describe where and how computers are used to process data and run applications. There are three main types:
Traditional Computing:
• This is the classic way of using computers.
• Programs are installed directly on a personal computer or workstation.
• Users work on a single machine (like a desktop or laptop).
• Does not rely heavily on the internet.
Example:
• Using Microsoft Word installed on your laptop
• Playing offline games
• Saving files directly on your computer
Web-Based Computing
• This type of computing depends on the internet and online services.
• Applications run through a web browser.
• Data is stored online (cloud storage).
• Users can access systems from anywhere with internet.
Example:
• Google Docs (online word processor)
• Email like Gmail
• Online learning platforms
Key idea: You don’t need to install everything—just use the internet.
Embedded Computing
• This is computing that is built inside machines or devices to control them.
• Computer systems are hidden inside devices.
• They perform specific tasks only.
• Usually not seen or directly controlled by users.
Example:
• Washing machines
• Microwaves
• Cars (engine control systems)
• Smart TVs
Key idea: Computers are inside everyday devices, not used like normal PCs.

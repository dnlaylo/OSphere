Module 1: Operating System Introduction
This module provides a broad introduction to the fundamental concepts of Operating Systems, covering how operating systems act as an intermediary between users and computer hardware, and how they manage system resources to ensure efficient and convenient computing. It explores the evolution of computing environments from early mainframe and batch systems to modern multiprogramming and time-sharing systems, as well as system architectures such as multiprocessor, distributed, clustered, real-time, desktop, and handheld systems. The module also discusses key OS goals and functions, including resource allocation, process management, memory and CPU scheduling, and system control, alongside different computing environments such as traditional, web-based, and embedded systems that shape how operating systems are used today.
Overview:
• It is  a program(software) that acts as an intermediary(interface) between a user of a computer and the computer hardware.
• It provides an environment in which a user may execute a program.
What is an Operating System?
1. Hardware – provides basic computing resources (CPU, memory, I/O devices).
2. Operating system – controls and coordinates the use of the hardware among the various application programs for the various users.
3. Applications programs – define the ways in which the system resources are used to solve the computing problems of the users (compilers, database systems, video games, business programs).
4. Users (people, machines, other computers). Computer System Components


a. Execute user programs and make solving user problems easier (by hiding the messy details which must be performed)
b. Make the computer system convenient to use (by presenting the user with a virtual machine, easier to use) 1. To provide a convenient environment:

Batch Systems
• A programmer creates a program using punched cards.
• The programmer brings the card deck to the IBM 1401.
• The IBM 1401 reads the cards and stores the jobs on an input tape.
• The operator carries the input tape to the IBM 7094 mainframe computer.
• The IBM 7094 processes the jobs and writes the results to an output tape.
• The operator carries the output tape back to another IBM 1401.
• The IBM 1401 reads the output tape and prints the results.
• The printed output is given to the programmer.

Purpose: To keep the mainframe (IBM 7094) focused on computation while slower input and output operations are handled by the IBM 1401 machines.

Multiprogrammed Batch Systems
• Job scheduling - the system must choose among several jobs to be brought into memory.
• Memory management – the system must allocate the memory to several jobs.
• CPU scheduling – the system must choose among several jobs ready to run.
• Allocation of devices. OS Features Needed for Multiprogramming
• Allows the CPU to process multiple programs
• While one program (job) is waiting for an I/O operation to complete, the CPU is switched to execute another program rather than remaining idle during I/O time.
• A program is resumed at the point where it was suspended when it gets its turn to use the CPU again.  Advantages of Multiprogramming
• Allow many users to share one machine simultaneously
• The CPU is multiplexed among several jobs that are kept in memory and on disk (the CPU is allocated to a job only if the job is in memory).
• A job is swapped in and out of memory to the disk.
• On-line communication between the user and the system is provided; when the operating system finishes the execution of one command, it seeks the next “control statement” from the user’s keyboard.
• On-line systems must be available for users to access data and code (instructions). Time-Sharing Systems–Interactive Computing
